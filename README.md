# Operon – System Activity Tracker (Windows)

## Simple Productivity & Activity Tracking for Windows

For release tagging and portable Windows builds, see [RELEASE.md](RELEASE.md).

---

## 1. What is this app?

System Activity Tracker is a lightweight Windows desktop app that helps you understand how your computer time is spent during the day.

It tracks:
- When you are actively working
- When the system is idle
- When the system is locked
- Which applications you use and for how long

All data is stored locally on your computer.
There is no cloud, no login, and no data sharing.

---

## 2. What does it track?

The app classifies time into three states:

- Active – keyboard or mouse activity is present
- Idle – system is unlocked but no activity beyond the configured idle threshold
- Locked – Windows session is locked

---

## 3. What can I see?

### Daily Summary
- Started time (first activity of the day)
- Ended time (last activity of the day)
- Total Active time
- Total Idle time
- Total Locked time

### Weekly Summary
- Week number
- Week start and end date
- Total Active / Idle / Locked time
- Day-by-day breakdown

### Monthly Usage
- Application-wise totals for the selected month
- Time spent actively in each app

### Application Usage
- List of apps used on the selected day
- Active time per application
- Sortable columns

---

## 4. Live Tracking Indicator

- Animated status indicator (Running / Stopped)
- Live Active timer
- Timer pauses automatically during Idle or Locked states

---

## 5. Controls

- Start Tracking
- Stop Tracking
- Refresh Summary
- Developer / Test Mode (for testing only)

---

## 6. Settings

- Idle threshold (minutes)
- Poll interval (seconds)
- Live refresh enable/disable
- Live refresh interval

Changes take effect only after clicking Save Settings.

---

## 7. System Tray Behavior

- Closing the window does not stop the app
- App continues running in the system tray
- Exit only via:
  - Header Exit button
  - Tray menu → Exit

---

## 8. Privacy

- Local-only storage
- No internet usage
- No telemetry
- No cloud sync

---

## 9. Who is this for?

- Personal productivity tracking
- Understanding daily work patterns
- Developers, students, and professionals
